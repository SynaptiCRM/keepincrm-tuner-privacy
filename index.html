2. Призначення розширення

Розширення кастомізує інтерфейс карток KeepinCRM на доменах *.keepincrm.com: угоди, клієнти, задачі. Корпоративні співробітники авторизуються через Sign in with Slack (робочий простір JustTech) — безпечний OAuth2-потік через дозвіл Chrome identity. Це підтверджує належність користувача до робочого простору компанії та надає авторизованим працівникам доступ до закритого функціоналу кастомізації CRM. Розширення не збирає паролі і не має форм реєстрації з паролем.

4. Куди передаються дані

Дані авторизації, heartbeat та telemetry передаються виключно для забезпечення роботи UI-тюнера:

  • https://slack.com — OAuth2-авторизація під час входу через Slack (користувач автентифікується безпосередньо в Slack; розширення отримує лише authorization code)
  • https://webhooks.justschool.com.ua/webhook/v1/keepintuner (HTTPS) — обмін OAuth code на сесію, heartbeat, telemetry
  • wss://webhooks.justschool.com.ua/justhook/v1/platform/keepintuner/ws (WSS) — realtime-події, наприклад відкриття угоди

Дані не продаються третім сторонам і не використовуються для реклами. Політика конфіденційності Slack: slack.com/trust/privacy/privacy-policy.

5. Шифрування та зберігання

  • Передача персональних даних — лише через HTTPS/WSS.
  • OAuth2-потік виконується через chrome.identity.launchWebAuthFlow; паролі не зберігаються в розширенні.
  • JWT та налаштування зберігаються локально в профілі Chrome користувача.
  • Терміни зберігання на сервері визначаються внутрішньою політикою JustSchool; за запитом користувача акаунт може бути видалений.

6. Режим Incognito

Розширення використовує режим incognito: spanning: стан авторизації та налаштування спільні між звичайним та приватним вікном браузера в межах одного профілю Chrome.
