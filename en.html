2. Purpose of the Extension

This extension customizes the KeepinCRM card interface on *.keepincrm.com domains (deals, clients, tasks). Corporate employees authenticate through Sign in with Slack (the JustTech workspace) — a secure OAuth2 flow using the Chrome identity permission. This verifies membership in the company workspace and grants authorized staff access to closed CRM customization features. The extension does not collect passwords and has no password-based sign-up forms.

4. Where Data Is Sent

Authentication, heartbeat, and telemetry data are transmitted solely to operate the UI tuner:

  • https://slack.com — OAuth2 authorization during Sign in with Slack (the user authenticates directly with Slack; the extension receives only an authorization code)
  • https://webhooks.justschool.com.ua/webhook/v1/keepintuner (HTTPS) — OAuth code exchange, heartbeat, telemetry
  • wss://webhooks.justschool.com.ua/justhook/v1/platform/keepintuner/ws (WSS) — real-time events such as opening a deal

Data is used solely to provide extension functionality. It is not sold to third parties and is not used for advertising. Slack privacy policy: slack.com/trust/privacy/privacy-policy.

5. Security and Retention

  • Personal data in transit is encrypted via HTTPS and WSS.
  • OAuth2 runs through chrome.identity.launchWebAuthFlow; passwords are not stored in the extension.
  • JWT tokens and UI settings are stored locally in the user’s Chrome profile.
  • Server-side retention follows JustSchool internal policy; account data can be deleted on request.

6. Incognito Mode

The extension uses incognito: spanning. Authentication state and UI settings are shared between regular and Incognito windows within the same Chrome profile, so the tuner works consistently in both modes.
